# UI / Backend Mapping Reference

Last updated: 2026-06-30

This file documents the current integration contract between `TransitNova.UI` / `TransitNovaUI.BusinessLayer` and `TransitNova.Api`.

## Endpoint Mapping Status

Current backend endpoint count:

```text
119 endpoints
```

Central UI endpoint catalog:

```text
UI/TransitNovaUI.BusinessLayer/ApiContracts/ApiRoutes.cs
```

`ApiRoutes.All` contains one route entry for every backend endpoint. Routes are stored as real UI-callable URLs:

```text
api/v1/...
```

ASP.NET route constraints are intentionally removed from UI routes:

```text
Backend route template: api/v{version:apiVersion}/users/shipments/{shipmentId:guid}
UI route:              api/v1/users/shipments/{shipmentId}
```

Use `ApiRoutes.Build(...)` to replace route placeholders safely:

```csharp
var route = ApiRoutes.Build(
    "api/v1/users/shipments/{shipmentId}",
    ("shipmentId", shipmentId));
```

## Response Envelope

Controller responses generated by `ResultExtensions` use this JSON shape on success:

```json
{
  "success": true,
  "message": null,
  "statusCode": 200,
  "data": {}
}
```

Failure response:

```json
{
  "success": false,
  "statusCode": 400,
  "errorCode": "SomeErrorCode",
  "message": "Readable error message"
}
```

Validation response:

```json
{
  "success": false,
  "statusCode": 422,
  "errors": [
    {
      "code": "Validation",
      "message": "Validation message"
    }
  ]
}
```

UI contract:

```text
UI/TransitNovaUI.BusinessLayer/ApiContracts/ApiResponse.cs
```

`ApiResponse.Errors` is now `IReadOnlyList<ApiErrorItem>?` to match the backend validation response array.
